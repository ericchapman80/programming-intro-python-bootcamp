# Instructor Materials

This folder is for public-safe reviewer guidance.

## Teaching Stance

This bootcamp is AI-aware on purpose.

We are not trying to ban AI, and we are not trying to let AI complete the project. The facilitator's job is to keep AI in the role of tutor, coach, debugger, and reviewer. Riley should still type code, run commands, read errors, make mistakes, explain decisions, and complete live changes.

> [!TIP]
> Think of AI like a patient instructor beside the student: it can ask better questions, explain a clue, or suggest the next small test. It should not take the keyboard away.

For the full AI policy, use:

- [AI Guidelines](../AI_GUIDELINES.md)
- [AI Coaching Guide](../docs/ai-coaching-guide.md)
- [AI Tooling Landscape](../docs/ai-tooling-landscape.md)

Do include:

- Coaching prompts
- Review checklists
- Demo checklists
- Rubrics
- Common mistakes
- Feedback language

Do not include:

- Full phase answer keys
- Private student information
- Complete hidden solutions

Full solution notes should live in a separate private instructor repository if needed later.
