# Phase 00 Review Checklist

Use this checklist while reviewing the student's Phase 00 pull request and demo.

## Expected Deliverable

The student should create:

```text
reflections/phase-00-reflection.md
```

The file should include brief notes about:

- What they learned
- Commands they used
- Bugs or mistakes
- AI usage
- Demo notes

## Commands The Student Should Be Able To Run

```bash
pwd
ls
cd
brew --version
python3 --version
git --version
code --version
git status
git switch -c phase-00-setup
git add reflections/phase-00-reflection.md
git commit -m "Complete phase 00 reflection"
git push -u origin phase-00-setup
```

The student does not need to have every command memorized.

## Questions To Ask

- What folder are you in right now?
- How can you tell?
- What is the difference between Terminal and VS Code?
- What is the difference between Git and GitHub?
- What does a commit do?
- What does a push do?
- What is a branch?
- What changed in this pull request?
- Did you use AI? What did it help you understand?

## Common Mistakes To Watch For

- Student is not inside the repository folder before running Git commands.
- Student clones the repository multiple times.
- Student confuses Git with GitHub.
- Student thinks pushing and committing are the same thing.
- Student cannot find the reflection file in VS Code.
- Student used AI but cannot describe what AI helped with.

## Good Enough To Move On

Move on when the student can:

- Open the repo in VS Code.
- Run basic Terminal commands.
- Explain the purpose of each tool in simple language.
- Show a committed reflection file on a branch.
- Explain what a pull request is asking for.
- Make one small live reflection edit without AI.

## Suggested Feedback Language

```text
You are ready for Phase 01 because you can find the project, open it in VS Code, run the basic commands, and explain the local-to-GitHub workflow.
```

```text
Before moving on, let's practice the difference between commit and push one more time.
```

```text
Your reflection is enough for Phase 00. Keep using exact command output when asking for help.
```
