# Phase Review Checklist

Use this checklist during each phase review.

- [ ] Student can run the code.
- [ ] Student can point to the files that changed.
- [ ] Student can explain the main concept.
- [ ] Student can explain one bug, mistake, or confusing moment.
- [ ] Student disclosed AI usage.
- [ ] Student can explain AI-assisted code.
- [ ] Student completed the PR template.
- [ ] Student can make a small live change without AI.
- [ ] Work is good enough to move on.

## Feedback Language

Helpful:

```text
You are ready to move on because the code runs and you can explain the main idea.
```

```text
Before moving on, explain this function in your own words and make one small change to it.
```

```text
This looks like AI may have written more than you understand. Let's slow down and explain it line by line.
```
